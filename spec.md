# 项目
Animaker 动画魅客（序列帧可视化选帧与导出工具）

# 目标
为游戏/动画的序列帧资源提供本地可视化工作台，支持方便浏览、精确选帧、预览循环效果，并一键打包导出（含 GIF 与动画配置）。

# 功能
- 帧序列浏览：选择文件夹，按文件名排序加载所有图片，仅保留常见图片格式。
- 列表选帧：支持多选、全选/反选/清空、奇数/偶数/随机选、区间选、上滚/下滚微调，以及「普通选择 / 单击追加」两种点击模式。
- 动画预览：根据选中帧循环播放，支持调整帧率、暂停/播放、循环/一次，支持多种背景颜色与 Fit/x1/x2 等缩放模式，显示当前帧序号、文件名和原始尺寸。
- 统计信息：底部状态栏显示当前文件夹路径、已选帧数量、所选帧总大小、当前帧率与合成时长。
- 导出：将选中帧导出为 ZIP，支持重命名帧（按 `{name}_001` 递增）、生成 `frames.json`、可选生成 `{name}.gif` 预览动画。
- 智能首尾帧：基于 RGBA+SSIM 自动搜索最相似的一对首尾帧，给出多组候选区间，支持在列表中一键选中对应帧段并预览。
- Phaser 动画定义：在导出时可同时生成 Phaser 所需的动画 JSON 片段（帧率、重复次数、图集名、帧名格式可配置）。
- 合并动画 JSON：从多个 JSON 中读取动画节点，合并为统一结构的 `anims` 配置并导出，支持扫描文件夹、添加文件、过滤无效项。

# 输入
- 帧序列所在目录路径（必选）。
- 可选：导出 ZIP/GIF/JSON 的目标保存路径、动画名称、Phaser 动画参数等。

# 输出
- 帧序列打包 ZIP（包含重命名后的帧文件与 `frames.json`）。
- 可选：同名 GIF 预览动画（保持透明度，最长边自动缩放到 512）。
- 可选：Phaser 动画 JSON 片段与动画合并 JSON（`anims` 列表）。

# CLI示例
- 启动 GUI（uv）：`uv run animaker`
- 启动 GUI（just）：`just run`

# 非目标
- 不在命令行下提供纯文本/无界面的选帧与导出流程。
- 不负责对图片内容做编辑（只做筛选、预览和打包），不内置网络同步/上传能力。

1. 顶部栏增加「导出」按钮，点击后选择保存目录，保存一个zip包；默认打开的目录是当前帧动画的目录，默认保存文件名使用该目录的名称；

3. 需要打包进zip的内容有：

 - 生成的frames.json文件，见下格式说明
4. 导出按钮右侧增加「导出时重命名帧」，默认勾选。选中时，导出的帧按 {name}_001 的格式重命名，其中name是保存的zip包名称, 帧序号从1开始，补齐至3位数
5. 导出按钮最右侧增加「导出GIF」，默认不勾选；勾选时导出的zip中增加使用选中帧按指定帧率播放形成的GIF，命名为{name}.gif。name为zip包名称。导出的帧最长边若大于512px，那么先缩小到最长边512后再生成gif
 
### frames.json文件 格式说明

```json
{
    "name": "fish01", # 保存的zip文件名，不含后缀
    "fps": 16, # 帧率
    "frame_count": 2, # 总帧数
    "frames": [ # 选中的帧序列保存文件名
        "1.png",
        "2.png",
    ],
    "src_dir": "~/Downloads/a", # 动画帧的源目录
    "src_frames": [ # 选中的帧序列源文件名
        "1.png",
        "2.png",
    ],
    "create_at": "2026-01-01 00:00:00", # 打包系统时间
}
```

## 智能首尾帧

需求：在导入的n张序列帧中，找到最相似的两帧作为首尾帧，两帧之间的序列作为可循环的序列帧。这称为智能首尾帧。

### 输入与输出

| 类型 | 说明 |
|------|------|
| **输入** | 当前打开的帧序列（n 张图），按列表顺序；每帧需能提供 RGBA（无 alpha 时视为全不透明）。 |
| **输出** | 一对下标 (i, j)，i ≤ j；区间 [i, j] 为可循环片段；首帧 i、尾帧 j 为「最相似」的一对。UI 上表现为：列表选中 i～j，并刷新首尾帧预览。 |

### 可提取参数（默认值，后续可做到界面）

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `alpha_threshold` | int | 40 | 前景 Mask：alpha > 此值的像素计入比较，否则忽略（透明/半透明边缘）。 |
| `min_gap` | int | 10 | 最小帧间距：只考虑 j - i ≥ min_gap 的 (i, j)，避免选到相邻/过近帧。 |
| `compare_size` | int | 256 | 比较前将帧缩放到 compare_size×compare_size（灰度）。序列帧常见 500×500 左右，64 过小易丢结构，建议 128～256。 |
| `top_k` | int | 5 | 取 SSIM 最高的前 top_k 对 (i, j)，再从中选一（当前实现可先只取 top 1）。 |
| `use_period_prior` | bool | False | 是否用周期先验：在 top_k 中优先选间隔接近预期周期的（暂不实现时可忽略）。 |
| `expected_period` | int \| None | None | 若 use_period_prior 为 True，期望的周期帧数（如 24）；None 表示不使用。 |

实现时先用上表默认值在代码里写死（或集中在一个配置结构里），界面不暴露；后续再把这些参数做到设置或弹窗输入。

### 实现思路：
#### 总体思路

这是一个“时间序列中的结构自相似搜索”问题。

核心技术：
 - Alpha 生成 Mask
 - 灰度结构表达
 - SSIM（结构相似度）
 - 帧间距约束（避免邻近帧误判）
 - 可选：周期先验优化

#### 实现步骤

##### 第一步：读取所有帧
按顺序加载 RGBA 图像。
保证分辨率一致。

##### 第二步：生成前景 Mask

从 alpha 通道生成 mask：
	•	alpha > 30（或 40）
	•	得到 bool mask

目的：

只比较鱼本体
忽略透明区域和边缘半透明噪声

技术点：NumPy 布尔数组

##### 第三步：结构表达简化

把 RGB 转为灰度图。

为什么？
	•	消除颜色波动影响
	•	强调形态结构
	•	SSIM 在灰度上更稳定

技术点：Pillow 转 L 或 skimage 灰度

##### 第四步：设定最小帧间距

避免 i 和 i+1 被选中。

比如：

min_gap = 10 或 12 帧

因为鱼摆动有周期，邻近帧必然很相似。

你要的是一个完整周期闭合点。

⸻

##### 第五步：计算结构相似度（核心）

对所有满足条件的 (i, j)：

只在 mask 区域计算 SSIM。

实现方式有两种：

方案 A（简单版）
裁剪到鱼的 bounding box
直接算 SSIM

方案 B（更严谨）
自定义 SSIM，仅统计 mask 区域像素

技术核心：

SSIM（Structural Similarity Index）

它衡量：
	•	亮度一致性
	•	对比度一致性
	•	结构一致性

范围 [-1, 1]
越接近 1 越相似

⸻

##### 第六步：排序候选帧对

选：
	•	SSIM 最大的前 3~5 组
	•	同时间隔接近预期周期更优

---

### 实现设计（按默认参数，先不暴露到界面）

1. **入参**：使用上表默认值（`alpha_threshold=40`, `min_gap=10`, `compare_size=256`, `top_k=5`；周期先验暂不实现）。
2. **流程**：  
   - 按顺序加载所有帧为 RGBA，统一缩放到 `compare_size × compare_size`，再转灰度。  
   - 用 `alpha > alpha_threshold` 生成每帧的前景 Mask（NumPy 或 Pillow 均可）。  
   - 只枚举满足 `j - i >= min_gap` 的 (i, j)，在 Mask 区域内计算两帧灰度图的 SSIM（先采用方案 A：用 Mask 的 bbox 裁剪两帧后算 SSIM；无前景时跳过该帧对）。  
   - 取 SSIM 最大的一对 (i, j)，作为返回值；UI 将列表选中设为 [i..j] 并刷新首尾帧。
3. **依赖**：使用成熟库，不自行实现。SSIM 使用 **scikit-image** 的 `skimage.metrics.structural_similarity`（需在 pyproject 中增加依赖）。Mask 与灰度用 Pillow + NumPy。
4. **后续**：参数表里的项在适当时机提取到设置面板或「智能首尾帧」弹窗的输入控件。

## 合并动画JSON数据

1. 增加「工具」菜单栏，下增「合并动画数据」，点击后弹出「合并动画数据」窗口
2. 该窗口布局参考「导出」窗口的布局，左侧是参数区域，右侧是动画数据json预览区，下方也是「取消、导出」按钮
3. 参数区域主体是一个列表，用于显示待合并的动画数据json文件绝对路径；列表下方依次有「扫描文件夹、添加、全选、清空」。
4. json预览区显示的内容，是通过读取列表选中的json文件，读取每一个json的anim节点合并为数组后，按```{"anims": [...anim数组]}```生成的合并后的json；每当选择项发生变化时，json及时刷新预览。
5. 「扫描文件夹」是打开一个文件夹并递归获取该文件夹下所有json文件（安全上限100个），「添加」是指单独选择一个json文件加入列表。「全选、清空」是对已有列表的快速选择。
6. 「导出」按钮将选择一个保存目录，默认保存文件名为anims.json

体验优化：
7. 「扫描」若超过100个文件则停止并弹出提示；「添加」允许一次选择多个json添加；忽略已经加入的文件（同路径），新加入的文件默认被选中。
8. 文件列表文字使用绿色/红色代表该文件是否是合法的json且有anim节点
9. 「全选」按钮右侧增加「仅合法」，点击后选中所有有合法节点的json
